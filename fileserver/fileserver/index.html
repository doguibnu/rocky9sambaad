    Servidor de Arquivos

    Membro e Servidor de Arquivos do AD

    Vamos configurar a Terceira máquina que servirá como servidor de arquivos do AD

    Iniciando o processo

    Update e Upgrade do sistema

    Faça um update e um upgrade do sistema. Utilize o comando:

    dnf update -y && dnf upgrade -y

    Instale o Samba e pacotes necessários

    Instale o samba e seus pacotes necessários para servidor de arquivos e também o editor de textos nano. Observe que iremos utilizar os pacotes disponíveis do Rocky Linux, sem precisar baixar e compilar o samba. Utilize o comando abaixo:

    dnf install -y nano samba samba-winbind samba-winbind-clients bind-utils

    Habilite os serviços smb nmb e winbind para iniciar junto ao boot do sistema:

    systemctl enable smb nmb winbind

    Configure e edite o arquivo /etc/hosts:

    Utilize o editor de texto nano para editar o arquivo /etc/hosts

    nano /etc/hosts

    127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
10.1.1.13   SRVFILE.seu.dominio SRVFILE
::1 localhost localhost.localdomain localhost6 localhost6.localdomain6

    Onde:

    10.1.1.13 = IP do seu Servidor de Arquivos

    SRVFILE.seu.dominio = nome completo do host+domínio

    SRVFILE = nome curto de seu host

    Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Edite o arquivo /etc/hostname

    Com o editor de textos nano edite o arquivo /etc/hostname com o comando:

    nano /etc/hostname

    srvfile.seu.dominio

    Salve o Arquivo

    Para salvar: control+o e para sair: control+x

    Editar o arquivo /etc/nsswitch.conf

    Com o editor de textos nano edite o arquivo /etc/nsswitch.conf com o comando:

    nano /etc/nsswitch.conf

    Localize as Linhas e Mude a configuração:

    De:
passwd: sss files systemd

Para:
passwd: files winbind


De:
group: sss files systemd

Para:
group: files winbind

    Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Configure a Rede

    Configure a rede e insira os dados do DNS sendo o IP do seu DC1 ou seja, do seu Controlador de domínio DC1. Execute o comando nmtui:

    nmtui

    Selecione Editar uma Conexão

    Digite Enter

    Com a tela Tab selecione Editar

    Com as teclas de direção (setas) mova-se até Servidores DNS e troque o IP para o IP de seu AD DC (controlador de domínio-DC1)

    Ainda com as setas de direção, selecione Domínios de pesquisa e insira: seu.domínio

    Então, selecione OK com as setas de direção e digite enter.

    Com a tecla Tab, selecione a opção: voltar e digite a tecla enter. Com a tecla Tab, selecione a opção OK e digite enter.

    Reinicie o sistema

    Execute o comando reboot para reiniciar o sistema

    reboot

    Arquivo user.map

    Crie e edite o arquivo user.map no diretório /etc/samba com o conteúdo:

    nano /etc/samba/user.map

    !root = DOMINIO\Administrator

    Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Backup do arquivo original smb.conf

    Crie um backup do arquivo /etc/samba/smb.conf com o comando:

    mv /etc/samba/smb.conf /etc/samba/smb.conf.bkp

    Verificar dispositivo de rede

    Veifique seu dispositivo de rede, pois o mesmo será inserido no arquivo que será criado. Execute o comando:

    ip a

    2: enp1s0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 52:54:00:fc:1d:33 brd ff:ff:ff:ff:ff:ff
    inet 10.1.1.41/24 brd 10.1.1.255 scope global noprefixroute enp1s0
       valid_lft forever preferred_lft forever
    inet6 fe80::5054:ff:fefc:1d33/64 scope link noprefixroute 
       valid_lft forever preferred_lft forever

    Seu dispositivo neste caso é o: enp1s0

    Crie e edite o arquivo smb.conf

    Vamos criar e editar o arquivo /etc/samba/smb.conf. Copie e coleo conteúdo para seu arquivo e não esqueça de alterar conforme suas informações. Execute o comando:

    nano /etc/samba/smb.conf

    [global]
    bind interfaces only = Yes
    interfaces = lo enp1s0 # sua interface
    dedicated keytab file = /etc/krb5.keytab
    kerberos method = secrets and keytab
    log file = /var/log/samba/%m.log
    min domain uid = 0
    realm = SEU.DOMINIODE       
    username map = /etc/samba/user.map
    security = ADS
    template homedir = /home/%U
    template shell = /bin/bash
    winbind refresh tickets = Yes
    winbind use default domain = Yes
    workgroup = DOMINIO
    idmap config dominio : range = 10000-999999
    idmap config dominio : backend = rid
    idmap config * : range = 3000-7999
    idmap config * : backend = tdb
    map acl inherit = Yes
    vfs objects = acl_xattr

    Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Backup do arquivo krb5.conf

    Faça uma cópia do arquivo /etc/krb5.conf com o comando:

    mv /etc/krb5.conf /etc/krb5.conf.bkp

    Crie um novo arquivo krb5.conf

    Crie e edite um novo arquivo krb5.conf e insira as linhas alterando para seu cenário. Utilize o comando:

    nano /etc/krb5.conf

    [libdefaults]
    dns_lookup_realm = false
    dns_lookup_kdc = true
    default_realm = SEU.DOMINIO

    Salve o arquivo

    Para salvar: control+oe para sair: control+x

    Comando testparm

    Verifique se existe algum erro de sintaxe com o comando:

    testparm

    Se não houver erros, execute o comando para recarregar o samba novamente:

    smbcontrol all reload-config

    Desabilite o SELinux

    Com o editor de textos nano edite e altere o arquivo 

    nano /etc/selinux/config

    Na linha `SELINUX=enforcing`
troque para:
SELINUX=disabled

Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Ajuste o Firewall

    Ajuste o Firewall, adicione o serviço do samba e recarregue o Firewall com os comandos abaixo:

    firewall-cmd --add-service=samba --permanent
firewall-cmd --reload

    Reiniciar o servidor

    Reinicie o servidor de arquivos com o comando:

    reboot

    Junte-se ao AD-DC1 (join AD)

    Utilize o comando abaixo para juntar-se ao seu Controlador de Domínio Principal DC1:

    net ads join -U administrator  
Enter administrator's password: senha do provisionamento

    Using short domain name -- DOMINIO  
Joined 'SRVFILE' to dns domain 'seu.dominio'

    Verifique a conexão NETLOGON

    Verifique a conexão Netlogon com o AD. Utilize o comando:

    wbinfo --ping-dc

    A saída do comando deve parecer como esta abaixo:

    checking the NETLOGON for domain[DOMINIO] dc connection to "DC1.seu.dominio" succeeded

    Nslookup

    Verifique se o seu DC1 (seu controlador de dominio principal) retorna com a resposta, executando o comando: nslookup SEU.DOMINIO. Altere os dados conforme seu cenário:

    nslookup DC1.SEU.DOMINIO

    Um resultado parecido como abaixo deve aparecer na tela:

    nslookup DC1.seu.dominio
Server: 10.1.1.31
Address: 10.1.1.31#53

Name: DC1.seu.dominio
Address: 10.1.1.31

    Também através do IP do seu DC1 com o comando:

    nslookup IP-serverad

    Antes de prosseguir, conecte-se em seu DC1 e crie um grupo e um usuário. No DC1 execute o comando para criar o grupo por exemplo:

    samba-tool group add 'Linux Domain'

    Logo após adicione um usuário com o comando:

    samba-tool user add 'Linux Test'

    Volte para o Servidor de Arquivos

    GID grupo a partir do Servidor de Arquivos

    Verifique se retorna o GID (número do Grupo) criado. Utilize o comando abaixo. A saída deve ser algo parecido como:

    getent group "DOMINIO\\Linux Domain"
linux domain:x:11115:

    Garantir Privilégios

    Garanta o privilégio de acesso para o usuário Administrator estar apto a ler/escrever/apagar dentro do disco. OBS: Altere o comando conforme seu cenário. Utilize o comando:

    net rpc rights grant "DOMINIO\Domain Admins" SeDiskOperatorPrivilege -U "DOMINIO\Administrator"
Enter DOMINIO\Administrator password: senha CRIADA no provisionamento do DC1
# Caso tenha criado corretamente o sistema irá dar sucesso do comando

Successfully granted rights.

    Recebendo a resposta como sucesso, está apto a iniciar a configuração de montagem de disco, o diretório escolhido para abrigar seus arquivos e ajustar as permissões de acesso, para que seja possível fazer a Administração através da Ferramenta RSAT do seu Windows.

    Por Exemplo:

    Caso tenha configurado um disco e disponibilizou/montou o diretório: /srv/ad, é necessário indicar esse caminho em seu arquivo smb.conf , que está no diretório: /etc/samba/smb.conf. Dentro do arquivo, localize a sessão global, indicando o nome de seu compartilhamento do servidor de arquivos, caminho e então ajuste as permissões de acesso seguindo o exemplo abaixo:

    OBS: Altere os dados conforme seu cenário

    nano /etc/samba/smb.conf

    [arqTeste]
   path = /srv/ad/
   read only = no
   browseable = yes

    Salve o arquivo

    Para salvar: control+o e para sair: control+x

    Ajustando Permissões

    Ajuste as permissões para o Grupo no diretório criado. Utilize os comandos:

    chown root:"Domain Admins" /srv/ad
chmod 0770 -R /srv/ad

    Recarregar o samba

    Recarregue novamente o smbcontrolcom o comando:

    smbcontrol all reload-config

    RSAT Windows

    Através do Windows, logue-se com a conta Administrator e através do RSAT inicie suas configurações desejadas.
